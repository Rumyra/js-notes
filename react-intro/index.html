Intro to react.js

It's not a framework - it does the UI very well - it's the V in MVC.

Component Driven

It's component driven, so it's all about creating HTML components that take state and properties and produce components. State can be changed, properties can not.

Components are optionally written in JSX, which means you can use HTML, rather than createElement The DOM representation in react is referred to as the virtual DOM, which is faster than the browsers DOM.

React computes the difference between whats in the DOM and what's been updated and just changes the updates - not the whole element

Instructions

  1. Include react.js and react-dom.js (CDN, node package or bower)
  2. Use external js file, rename extension to .jsx so react can parse as such
  3. Create DOM element for react to hook into
  4. To create component in react - use createClass method. Takes an object with properties - only required property is render
  5. Use React.render and pass in component name and root dom element.
